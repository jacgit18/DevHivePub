### Core Decision Criteria

1. **Atomicity Principle**  

   → **New Note:** If the idea is a *distinct, self-contained concept* that stands alone (e.g., "Benefits of Intermittent Fasting").  

   → **Merge:** If it’s a minor refinement or example of an existing concept (e.g., adding a personal experiment result to your "Nutrition Experiments" note).

  

2. **Topic Scope**  

   → **New Note:** Broad/new subject area (e.g., "Introduction to Quantum Computing").  

   → **Merge:** Nuances/subtopics of an established topic (e.g., adding "Deferred Measurement Principle" under your "Quantum Computing Fundamentals" note).

  

3. **Cross-Linking Potential**  

   → **New Note:** High potential for future links *to/from* multiple other notes (e.g., "Cognitive Load Theory").  

   → **Merge:** Only relevant within one parent topic (e.g., "Meeting notes with Team X" under "Project Y Documentation").

  

4. **Future Discoverability**  

   → **New Note:** If you’ll likely search for this idea *independently* later.  

   → **Merge:** If it’s only meaningful in context (e.g., a book highlight that only makes sense within your "Book: Deep Work" summary).

  

5. **Note Size Threshold** (Personal Rule)  

   → **New Note:** When an existing note exceeds ~500 words *and* new content introduces a separable theme.  

   → **Merge:** When additions are <150 words and deepen existing context.

  

---

  

### Workflow Triggers

- **Create NEW When:**  

  - You reference the idea 3+ times across other notes.  

  - It bridges 2+ unrelated topics (e.g., "Psychology of Habit Formation" connecting health + productivity notes).  

  - It’s a template (e.g., "Meeting Template", "Project Retrospective").  

  

- **MERGE/Integrate When:**  

  - It’s a direct response/update to existing content (e.g., adding counter-arguments to your "AI Ethics" note).  

  - Fragments (<3 sentences) that enrich without standing alone.  

  - Redundant ideas (merge duplicates).  

  

---

  

### Hybrid Strategies

1. **Temporary "Staging" Notes**  

   → Collect related fragments in a draft note (e.g., `zz_temp_AI_ideas.md`), then merge into final notes weekly.  

2. **Hub Notes**  

   → Create a "Hub" (e.g., `Productivity_Systems.md`) linking to atomic child notes (Zettelkasten-style).  

3. **Aliases/MOCs**  

   → Use `[[alias]]` or "Map of Content" notes to group related notes *without* merging files.  

  

---

  

### Warning Signs to Fix

- **Over-Fragmentation:** >5 notes on the same topic with <100 words each.  

- **Over-Consolidation:** Notes >1500 words covering 5+ unrelated ideas.  

- **Orphan Notes:** Notes with 0 backlinks after 30 days (consider merging/deleting).  

  

### Pro Tips

1. **Review Monthly:** Audit notes with tools like Obsidian's Graph View or `grep` for orphaned content.  

2. **Tag by Status:** `#seedling` (new/fragile), `#evergreen` (mature), `#archive` (merged/obsolete).  

3. **Force Atomic Titles:** If you can’t title it clearly in 5-7 words, it’s likely not atomic.  

  

> **Balance Philosophy:**  

> *"New notes maximize flexibility; merging maximizes depth. Optimize for retrieval, not creation speed."*  

  

This system prioritizes **long-term knowledge evolution** over short-term organization. Adjust thresholds based on your recall needs!
