Non-functional requirements specify the qualities or attributes that the system should possess to ensure its overall performance, security, reliability, usability, and scalability. Unlike functional requirements that focus on what the system should do, non-functional requirements address how the system should perform or behave under various conditions.

Example non-functional requirements:

1. **Performance**:
   - Response time for user interactions should be less than 500 milliseconds under normal load conditions.
   - The system should maintain consistent performance even during peak usage periods.

2. **Scalability**:
   - The system should be able to accommodate a minimum of 100,000 concurrent users without significant degradation in performance.
   - Scalability should be achieved through horizontal scaling of resources and load balancing techniques.

3. **Security**:
   - Data should be encrypted both in transit and at rest using industry-standard encryption algorithms (e.g., TLS for data in transit, AES for data at rest).
   - User authentication and authorization mechanisms should be implemented securely to prevent unauthorized access to sensitive information.

4. **Reliability**:
   - The system should have a system uptime of at least 99.9%, ensuring high availability for users.
   - Failover mechanisms and redundancy should be in place to minimize downtime in the event of system failures.

5. **Usability**:
   - The user interface should be intuitive, responsive, and accessible to users with disabilities.
   - User documentation and help resources should be provided to assist users in navigating and using the system effectively.

These non-functional requirements define the attributes that are crucial for the system's overall quality, effectiveness, and user satisfaction. They provide criteria for evaluating the system's performance, security, reliability, usability, and scalability.
